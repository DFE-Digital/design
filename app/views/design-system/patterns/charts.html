Visual elements

Using charts can introduce difficulties for users with cognitive or visual impairments.

Think about whether users with different needs will understand the information your chart represents.

Colour and contrast

Your data must not rely on colour differences to be understood. Users with conditions like colour blindness may find this difficult.

Avoid using:

  • patterns or textures
  • shaded backgrounds
  • shadows

WCAG (Web Content Accessibility Guidelines) describe the minimum contrast levels content must have.

Item spacing

Include spaces between bars and other elements of your chart to make them easier to compare.

Gaps between bars should be narrower than a single bar, but the correct spacing will depend on your situation.

You should test this with your users.

Fixed images

Use the SVG file format if you embed pictures that represent data. These scale when zoomed and can be easier to read.

You must include a text alternative for fixed images.

Text alternatives

Your charts must also have at least one text version of the same information. The type you choose should fit how you expect your chart to be used.

If users only need the data

Where users are likely to understand the context of your chart and only need to see the data it contains, use a table in an accessible format.

If context is needed or if there's a message

Use a text description if you need to describe what your chart is about or to summarise what it shows.

A text description should:

  • be in the main body copy
  • describe the context
  • outline what the data shows

A text description should not:

  • be place inside alt text (alternative text)
  • repeat the chart title
  • describe each element
  • go into detail about data

You can use a written description and a table together.

Alt text

If your chart is a static image, it must also have an alt text description.

Alt text should:

  • explain the chart type
  • describe the information it contains
  • be separate from the written description

Titles

Your chart must have at least a headline title, but it's better if it also has a statistical subtitle.

The titles should be within the body text and not as images of text.

Headline title

This should communicate the headline message of the chart.

For example: 'Increase in the number of people completing apprenticeships'.

Statistical subtitle

A statistical subtitle should say:

  • what the data is
  • the geography it relates to
  • the time period shown
For example: 'Figure 1: Number of people completing an apprenticeship, England, 1994 to 2024'.

Keyboard navigation

Make sure that keyboard users can use the tab key to navigate into your chart and interact with it as a mouse user would be able to.

Test your charts are accessible

Install and run accessibility testing tools to help you spot problems with your visual data.

You can also check your finished designs against the Government Analysis Function checklist.

More help

You can see more detailed information about accessible charts on the Government Analysis Function data visualisation pages.
