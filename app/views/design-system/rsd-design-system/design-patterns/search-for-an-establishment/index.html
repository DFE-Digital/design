Show information of an establishment step-by-step.

When to use this pattern

When users need to search for information about an establishment (e.g. an academy trust or a school), or provide details about the establishment.

How it works

There could be a few different scenarios:

  1. searching for an establishment as read-only information
  2. searching for an establishment as part of a longer journey

1. Searching for an establishment as read-only information

You may want to search for information of an establishment for read-only purpose without adding the information retrieved into another user journey.

Basic Design

For the basic version where JavaScript is disabled, the pages involved should be:

  1. Provide a simple input box for keyword search.
  2. Provide a list of possible results, ideally in an overview format.
  3. Provide the detailed information of the result selected.

Progressive enhancement

For the progressive enhancement where JavaScript is allowed, the pages involved should be:

  1. Provide a input box with autocomplete enabled for keyword search. Show the list of possible results under the input box dynamically.
  2. Provide the detailed information of the result selected.

2. Searching for an establishment as part of a longer journey

You may want to search for information of an establishment and use it as part of a longer journey.

Basic Design

For the basic version where JavaScript is disabled, the pages involved should be:

  1. Provide a simple input box for keyword search.
  2. Provide a list of possible results, ideally in an overview format, and allow users to choose the which option they want to see the detailed information in the next page.
  3. Provide the detailed information of the result selected. Ask users to confirm that this is the establishment they want, or bring them back to the search results page if the result is not what they want.

Progressive enhancement

For the progressive enhancement where JavaScript is allowed, the pages involved should be:

  1. Provide a input box with autocomplete enabled for keyword search. Show the list of possible results under the input box dynamically.
  2. Provide the detailed information of the result selected. Ask users to confirm that this is the establishment they want, or bring them back to the keyword search page if the result is not what they want.

Research on this pattern

We have not done research on this design pattern, but the pattern is being used in Prepare conversions and transfers and Complete conversions, transfers and changes to meet this need from our user needs database (opens a DfE Microsoft List):

User need 22: when I have a question about project data in MAP Complete and Prepare, I need the name and email address of the DO or RCS caseworker responsible for the project, so I can get in touch with them to address my query.
