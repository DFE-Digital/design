Users frequently include information about where they have sourced their information from when producing internal documents. Sometimes, dates can often be misinterpreted when given without further context of what the information it is refering to.

For example, confusion between when information was last updated at the source, versus when it was imported from an external system.

It's important to display sources and dates clearly and simply, to avoid confusion as much as possible. Sometimes, dates and sources are used when the credibility of information is in question e.g. at a board meeting.

When to use this pattern

This pattern should be used when:

  • information is automatically taken from an external service or product and displayed on a page
  • information is edited directly within a service or product by a human

When not to use this pattern

This pattern should not be used when:

  • information is automatically taken from a different page within the same service or product

How it works

A single source and its last updated date can be displayed in the following way:

[Page title]:

  • [Information type] was taken from [Source name] on [Date]

When information comes from the same source

Each entry follows the same grouping as how it is presented on the page.
