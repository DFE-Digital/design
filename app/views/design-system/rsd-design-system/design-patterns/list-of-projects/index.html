When to use this pattern

Provide a view where your users can see a list of the summary of projects in your service.

For a lot of internal facing case management services, it is very often that our service will need to provide a view for our users to see multiple projects at a glance. Therefore, seeing a list of projects is a common user need.

Content for each project

Very often, there will be many fields in each project. It is impossible to put all the fields into a project record on the project listing page. Therefore, it is recommended to conduct user researches to make sure the fields you picked are the most useful for your users. Here are some common fields that could be included:

  • project title
  • project ID
  • key dates
  • project status
  • a link to the individul project page (could be the project title)

Project summary layout

Listing of projects should always be in a vertical format. But regarding the format of each project summary, there are 2 variations you can use, depending on the context:

  1. tabular design
  2. card design

The choice of design depends on your services. Common factors to consider includes:

  • whether it is important for the users to be able to compare projects
  • whether it is important to see a lot of information in the summaries
  • page width

Research on this pattern

We have not done research on this design pattern, but parts of the pattern are being used in Find information about academies and trusts and Manage free school projects to meet this need from our user needs database (opens a DfE Microsoft List):

User need 8: When managing my team's workload, I need to see the projects each individual is working on alongside new incoming projects, so that I can understand capacity in my team and allocate project accordingly.

User need 30: When looking at the projects happing in my region, I need to know the status and progress being made on different projects so that I see the workload of different team members.

User need 90: When accessing Prepare and looking for my projects, I need an easy way to view all projects I'm working on in one view, so I can monitor and manage my own workload.
