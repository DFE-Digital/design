When to use this pattern

Use this pattern when users need to see summary information about a list of projects.

Summary fields for each project

When choosing which fields to display for each project, research with your users to see which fields they find most useful.

Here are some fields you could display:

  • project title
  • project ID
  • dates
  • project status
  • a link to the individual project page. This could be the project title

List format

Display the list of projects in a vertical format.

There are 2 variations you can use:

  1. tabular design
  2. card design

Choose the variation that works best for your service. Consider page width and whether it's important for users to:

  • compare projects
  • see a lot of information about a project up-front

Card design

A card design uses the GOV.UK Table component to stack up projects, but the content and layout inside each project is more flexible.

The height of each project summary is relatively tall. Therefore the card design is suitable if users need to read lots of information about a project at a high level, but comparing projects is not important.

Research on this pattern

We have not done research on this design pattern, but parts of the pattern are being used in Find information about academies and trusts and Manage free school projects to meet this need from our user needs database (opens a DfE Microsoft List):

  • user need 8: when managing my team's workload, I need to see the projects each individual is working on alongside new incoming projects, so that I can understand capacity in my team and allocate project accordingly
  • user need 30: when looking at the projects happing in my region, I need to know the status and progress being made on different projects so that I see the workload of different team members
  • user need 90: when looking for my projects, I need an easy way to view all projects I'm working on in one view, so I can monitor and manage my own workload
