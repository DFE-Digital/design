When to use this component

Use this pattern to create a product footer that is consistent with other RSD products.

When not to use this component

On

  • external-facing sites, like Apply to become an academy (A2B) This site is for external users and the Microsoft links in this pattern won't work for them
  • SharePoint guidance pages, like Prepare conversions and transfers - SharePoint

How to use this component

You can copy the HTML above and paste it where you'd like to display the footer.

Get help

It has a 'Get help' h2 with two links underneath it:

Email Service Support for help with using this system

This is a mail-to link with the subject header:

[Product name]: support query

For example, Prepare conversions and transfers: support query

It has this text underneath the link: "We aim to respond within 5 working days, or one working day for more urgent queries."

How to use this system (opens in a new tab)

This links to the section of a product's SharePoint page with guidance on how to use the product. For example, Guidance for using Prepare conversions and transfers.

If you don't have SharePoint guidance about how to use your product, you do not need to include this hyperlink.

Give feedback

It also has a 'Give feedback' h2 with two links underneath it:

Give feedback (opens in a new tab)

This links to a Microsoft Form where users can give feedback about the product. The same Microsoft Form is used across all RSD products: Give feedback about a Regions Group system - Microsoft Form.

Suggest a change (opens in a new tab)

This links to a Microsoft Form, which is the same across all products: Suggest a change to a Regions Group system - Microsoft Form.

Other links

  • an 'Accessibility statement' link which goes to the product's accessibility statement
  • a 'Cookies' link which goes to the product's cookies page
  • a 'Privacy notice' link which goes to the product's privacy policy. If the product does not need a privacy notice, you do not need to include this link
  • Crown copyright link, with the Crown logo

Research on this component

We have not done research on this component, but it is being used to meet these needs from our user needs database:

  • user need 65 - when something I need is missing from a product, I need to give feedback to the digital team so that my need can be met
  • user need 72 - when I am having trouble using the service, I need to get help so that I can resolve my issue and get back to work
