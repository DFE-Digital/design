Prototyping {% markdown %} # {{ title }} You will need a dedicated folder to store all prototypes on your computer. ## Create a folder Open VS Code and from the toolbar, select Terminal > New Terminal. Run the following command to create a new folder: ### For Windows

Code

mkdir C:\Users\%USERNAME%\Documents\prototypes
cd C:\Users\%USERNAME%\Documents\prototypes
### For Mac

Code

mkdir ~/Documents/prototypes 
cd ~/Documents/prototypes
> **mkdir** means 'make directory' and creates a new folder in the location you specify. > > **cd** means 'change directory' and will change the terminal to work in the new folder you have just created. ## Create a new prototype We're now going to create a new prototype called 'my-prototype'. This will create a new folder with this name, in the prototypes folder you just created. In the terminal, run the following command to create a new prototype:

Code

npx govuk-prototype-kit@latest create my-prototype
Node.js makes use of packages which are self-contained code that are built do perform a specific function, and can be easily shared and reused. Node comes with a package manager called npm (node package manager), which is used to install packages from the [npm registry](https://www.npmjs.com/). > **npx** is a tool that comes with npm. It is used to execute packages from the npm registry. > > **govuk-frontend/prototype-kit@latest** is the package name for the GOV.UK Prototype Kit and the @latest tag tells npm to install the latest version of the package. > > **create my-prototype** is the command that tells the prototype kit to create a new prototype called 'my-prototype'. Next, you want to change your terminal to work with the new prototype directory. Run the following command in the terminal:

Code

cd my-prototype
Now we want to run the prototype in development mode. This will start the prototype running on your device so you can view it in the browser. Run the following command in the terminal:

Code

npm run dev
You should see a message in the terminal which says: 
The Prototype Kit is now running at: 
http://localhost:3000
You can now view your prototype in the browser by going to this address. > **npm run** is a command that tells npm to run a script that is defined in the package.json file in the root of your prototype folder. > > **dev** is the name of the script that tells the prototype kit to run the prototype in development mode. Prototypes run on ports which are used to identify different web services running on the same device. The prototype will run on port 3000 by default. This is the address you will use to view your prototype in the browser. > **localhost** is a hostname that refers to the device you are using. It is used to access the device you are working on, and is not accessible to anyone else. > > **3000** is the port number that the prototype is running on. This is the address you will use to view your prototype in the browser. ## Stopping your prototype from running You might need to stop your prototype from running if you want to run a different prototype, or if you need to stop working on your prototype for a while. To stop your prototype from running, press `Ctrl + C` in the terminal. This will stop the prototype from running and return you to the command line. If you need to start your prototype again, rerun the `npm run dev` command in the terminal. > **Hint** > > You can also use the up and down arrows to cycle through previous commands you have run in the terminal. If you don't see the message about the prototype running, you may need to check the terminal for any error messages. You can check the [troubleshooting](/design-system/prototyping/tips-troubleshooting) guide for help with common issues.
