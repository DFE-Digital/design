Prototyping {% markdown %} # {{ title }} {{ description}} You must have a GitHub account to work through this section. See the [pre-requisites](/design-system/prototyping/prerequisites#github-access) section for more information. ## Create a new repository To create a new repository in GitHub, follow these steps: 1. If you aren't signed in already, sign in to your GitHub account at [GitHub](https://github.com) 2. Go to the [DfE-Digital](https://github.com/dfe-digital) organisation page 3. Click on the 'New' button You'll be taken to a page where you can create a new repository. For the purpose of this guide, use your name followed by '-prototype'. For example, 'jane-doe-prototype' > Prototypes should have obvious names followed by '-prototype'. > > Always use lowercase letters and hyphens instead of spaces. 4. In the 'Repository name' field, enter a name for your repository 5. Make your repository private, unless you're working on an open source project, or you're happy for anyone publicly to see your code 6. Click the 'Create repository' button You've now created a new repository in GitHub where you can store your prototype code. You should see some information about how to clone or push to your repository. Keep this page open as you'll need some information off it. ## Add your code to the repository To add your code to the repository, we're going to use the terminal in VS Code to associate your local code, with the repository you've just created. If your prototype is running, stop the server by pressing `Ctrl + C` in the terminal. ### Copy your repository URL From the GitHub page, copy the whole line which starts with **git remote add origin** from the 'or push an existing repository from the command line' section and paste it into the terminal and press enter or return. This will associate your local code with the repository you've just created. Next we're going to add all the files into the main branch of the repository. In the terminal, run this command:

Code

git branch -M main
> This command renames the default branch from `master` to `main`. This is a more inclusive term and is the default branch name for new repositories on GitHub. > > You only need to run this command once when working with a new repository. You're now ready to add your code to the repository. In the terminal, run this command:

Code

git push -u origin main
This command will push your code to the repository you've just created. You should see some information about the files being pushed to the repository. > **push** is the command to send your code to the repository > > **-u** is the command to set the upstream repository > > **origin** is a shorthand name for the remote repository on GitHub, it saves you having to type the full URL > > **main** is the name of the branch you're pushing to. when pushing changes in the future, you can just use the command `git push` to send your code to the repository. You've now added your code to the repository in GitHub. If you go back to the GitHub page and refresh it, you should see your code in the repository.
